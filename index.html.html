Stage 0 – Data Ingestion & State Builder

Goal

Prepare consistent, analysis-ready time-windowed state.

Implementation Details

  • Entry point: main.py
  • Aggregation logic: core/state_builder.py
  • Config-driven metadata: utils/config.py
  • Date grain: Day
  • Rolling window: 14 days
  • Feature-specific aggregation rules

Key Concept

Raw warehouse data → grouped by date → aggregated per feature rules → standardized state vector.

Raw Data → Config Mapping → Aggregation → Windowed State

Stage 1 – Multivariate Deviation Detection

Core Technique: Hotelling's T²

T²ₜ = (xₜ - μ)ᵀ S⁻¹ (xₜ - μ)

μ = historical mean vector | S = covariance matrix | Threshold = Chi-square critical value

Signals Computed

  • T² score
  • Persistence score
  • Risk zone score
  • Exceedance count
  • Weighted Risk Score

Outputs

  • Anomaly Level: none / marginal / moderate / strong
  • Decision Gate: stop / proceed
  • KPI contribution breakdown

Not just statistical deviation — governed anomaly classification.

Stages 2 & 3 – Context + Causal Hypothesis

Stage 2 – Contextual Embedding

  • Determines direction (↑ / ↓ / flat)
  • Computes normalized strength
  • Converts deviation → interpretable signals

Stage 3 – Bayesian Causal Reasoning

For each hypothesis:

Posterior = Prior × Likelihood

  • Direction match
  • Signal strength
  • Soft penalty (0.2) if mismatch
  • Likelihood floor enforcement

Statistical Anomaly → Probabilistic Explanation

Ranked hypotheses with normalized posterior probabilities

Stages 4 & 5 – Narrative + Decision Signals

Stage 4 – LLM Narrative Synthesis

  • Triggered only if anomaly validated
  • Extract top hypothesis
  • Use materially changed features (> 0.3 strength)
  • Generate 3-sentence executive summary
  • Fallback if LLM fails

LLM is gated and evidence-based

Stage 5 – Decision KPIs

  • NCI → Max posterior probability
  • DER → Entropy reduction
  • CDS → Causal dominance score

These quantify: Confidence, Uncertainty reduction, Strength of single dominant cause

Strengths & Trade-offs

✓ Strengths

Explainable

Every anomaly comes with causal reasoning and narrative explanation

Modular

Pipeline stages are independent and can be enhanced individually

Config-driven

Feature definitions and thresholds externalized for easy tuning

Business-aligned

Thresholds and outputs designed for operational decision-making

LLM safely gated

Narrative generation only occurs after statistical validation

⚠ Trade-offs

Prior hypothesis required

System needs pre-defined causal hypotheses; cannot discover unknown patterns

Baseline dependency

Performance heavily relies on quality and stability of historical baseline window

Entropy sensitivity

DER and related metrics can fluctuate with prior probability distribution choices
